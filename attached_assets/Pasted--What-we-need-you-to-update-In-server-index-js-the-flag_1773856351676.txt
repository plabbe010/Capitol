## What we need you to update
 
### In `server/index.js` — the flag engine
 
Find the function called `computeFlags` (or wherever flag logic lives) and replace
it with a two-tier system:
 
---
 
### 🔴 TIER 1: "alert" — genuinely suspicious, sorts to top of feed
 
Flag as `alert` if ANY of these are true:
 
1. **First time this member has purchased this ticker**
   - Check the full trades dataset — if there is no earlier trade where
     `representative === this member` AND `ticker === this ticker` AND it was
     a purchase, flag it.
 
2. **Bipartisan buy**
   - If this is a purchase, check if any member from the *opposite* party also
     purchased the same ticker within 30 days (before or after this trade date).
   - Reason string: `"Bipartisan signal — both D & R buying within 30 days"`
 
3. **Committee overlap with sector**
   - We have a committee map (member name → array of committees they sit on).
   - We also have a sector map (ticker → sector) and a sector→committees map.
   - If the member sits on a committee that oversees the stock's sector, flag it.
   - Reason string: `"On [committee name] — directly oversees this sector"`
 
4. **Filed at or near the 45-day legal limit**
   - Calculate gap = days between `transaction_date` and `disclosure_date`.
   - If gap >= 40 days, flag as alert.
   - Reason string: `"Filed [N] days after trade — pushed to the legal limit (45d max)"`
 
---
 
### 🟡 TIER 2: "flag" — worth watching, shown below alerts
 
Flag as `flag` if ANY of these are true (and none of the alert conditions fired):
 
1. **Large position**
   - If estimated trade value >= $250,000: reason = `"Large position — ~$[X] disclosed"`
   - If estimated trade value >= $50,000 AND it is more than 3× this member's
     average trade size: reason = `"3× larger than this member's typical trade"`
   - If estimated trade value >= $50,000 (baseline): reason = `"Notable size — ~$[X] disclosed"`
 
2. **Slow disclosure**
   - If gap between transaction and disclosure is 30–39 days.
   - Reason string: `"Slow to disclose — [N] days between trade and filing"`
 
3. **Obscure or small-cap ticker**
   - If the ticker is NOT in the known large-cap list AND has no known sector mapping,
     AND it is a purchase — flag it.
   - Reason string: `"Obscure or small-cap ticker — not a typical congressional pick"`
 
---
 
### Output format per trade
 
Each trade object sent to the client should include:
 
```json
{
  "flagTier": "alert" | "flag" | null,
  "flagReasons": ["Primary reason", "Secondary reason if any"]
}
```
 
- `flagTier` is `"alert"` if any alert condition fired, `"flag"` if only flag
  conditions fired, `null` if nothing fired.
- `flagReasons` lists all reasons that fired, alert reasons first.
- Trades should be sorted: alerts first, then flags, then normal — newest date
  within each group.
 
---
 
### In the feed sort order
 
After enriching trades with flag data, sort the array like this:
```
alert trades (newest first) → flag trades (newest first) → normal trades (newest first)
```
